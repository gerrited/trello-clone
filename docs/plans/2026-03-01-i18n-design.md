# Design: UI Internationalization (i18n)

**Date:** 2026-03-01
**Status:** Approved
**Scope:** Full UI coverage — DE, EN, IT, FR

---

## Overview

Add multi-language support to the trello-clone UI using i18next + react-i18next. Language preference is saved per-user in the database and also persisted in localStorage for unauthenticated users. A language selector appears on the login/register pages and in the user menu.

---

## Supported Languages

| Code | Language |
|------|----------|
| `en` | English  |
| `de` | Deutsch  |
| `fr` | Français |
| `it` | Italiano |

---

## Architecture

### Library

- `i18next` — core translation engine
- `react-i18next` — React bindings (`useTranslation`, `Trans`)
- `i18next-browser-languagedetector` — auto-detects from browser/localStorage

All packages added to `apps/web` only.

### Translation Files

Bundled JSON files (no HTTP backend — translations ship with the app):

```
apps/web/src/locales/
  en/translation.json
  de/translation.json
  fr/translation.json
  it/translation.json
```

Single namespace (`translation`), keys prefixed by feature area.

### i18next Config (`apps/web/src/i18n.ts`)

```ts
i18n
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    resources: { en, de, fr, it },
    supportedLngs: ['en', 'de', 'fr', 'it'],
    fallbackLng: 'en',
    detection: {
      order: ['localStorage', 'navigator'],
      caches: ['localStorage'],
      lookupLocalStorage: 'i18nextLng',
    },
    interpolation: { escapeValue: false },
  });
```

### Language Persistence

**Pre-login (unauthenticated):**
- `i18next-browser-languagedetector` reads browser `navigator.language` as default
- Persists selection to localStorage key `i18nextLng`
- Supported language codes: `en`, `de`, `fr`, `it`; falls back to `en` if unsupported

**Post-login (authenticated):**
- `authStore.setUser()` calls `i18next.changeLanguage(user.language)` after login/getMe
- DB language overrides localStorage on login
- When user changes language via UserMenu → API call to update profile → `i18next.changeLanguage()` immediately

## Backend Changes

### Database Schema (`apps/api/src/db/schema.ts`)

Add to `users` table:
```ts
language: varchar('language', { length: 5 }).notNull().default('en'),
```

Generate and apply Drizzle migration:
```bash
pnpm db:generate
pnpm db:migrate
```

### Shared Types (`packages/shared/src/types/user.ts`)

Add `language: string` to the `User` interface.

### Shared Validation (`packages/shared/src/validation/`)

**`updateProfileSchema`** — add:
```ts
language: z.enum(['en', 'de', 'fr', 'it']).optional(),
```

**`registerSchema`** (in `auth.schema.ts`) — add:
```ts
language: z.enum(['en', 'de', 'fr', 'it']).optional(),
```

### Service Layer

No significant changes needed:
- `updateProfile()` — pass through `language` field if provided
- `register()` — pass through `language` field if provided (defaults to `'en'` via DB default)
- `getMe()` — already returns full user object, no change needed

## Frontend Changes

### i18n Setup

`apps/web/src/i18n.ts` — new file with i18next config (see above)

`apps/web/src/main.tsx` — import `'./i18n'` before `<App />` renders

`apps/web/index.html` — remove hardcoded `lang="de"`, set dynamically:
```ts
i18n.on('languageChanged', (lng) => {
  document.documentElement.lang = lng;
});
```

### LanguageSelector Component

New component `apps/web/src/components/LanguageSelector.tsx`:
- Button group showing `DE | EN | IT | FR`
- Active language highlighted
- Same visual style as the theme selector (bordered button group)
- Calls `i18next.changeLanguage(code)` on click
- For authenticated users: also calls `updateProfile({ language: code })` API + updates authStore

### UI Placement

| Location | Placement |
|----------|-----------|
| `LoginPage.tsx` | Top-right corner of the login card |
| `RegisterPage.tsx` | Top-right corner of the register card |
| `UserMenu.tsx` | New row below the theme selector row, with globe icon |

### Language on Registration

`RegisterPage.tsx` sends `language: i18n.language` with the registration request so new users start with their chosen language saved to DB.

### authStore Integration

In `authStore.setUser()` (and wherever `getMe()` resolves), call:
```ts
import i18n from '../i18n';
i18n.changeLanguage(user.language);
```

### All Components

Every component with hardcoded strings uses `const { t } = useTranslation()` and replaces literals with `t('key')` calls.

## Translation Key Namespaces

All keys in a single `translation.json` per language, organized by prefix:

| Prefix | Coverage |
|--------|----------|
| `auth.*` | Login, register, form labels, OAuth buttons, validation errors |
| `user.*` | Profile modal, password change, menu items, language label |
| `theme.*` | Light, dark, system labels |
| `language.*` | Language selector label |
| `board.*` | Board page, column/swimlane/card actions, empty states |
| `card.*` | Card detail modal, assignees, labels, due date, description |
| `comment.*` | Comments section, add/edit/delete |
| `team.*` | Team management, member roles |
| `notification.*` | Notification panel, mark-read actions |
| `common.*` | Save, cancel, delete, loading, error messages, confirm dialogs |
